## 一、产品定位

**一句话**：曲库即素材库，从听歌无缝进入练习。

**MVP 聚焦**：为吉他初级学员（1-6 个月经验）提供 Web 端练习工具 + AI 陪练镜子，用 AI Cover 音频解决版权冷启动。

**AI 设计立场**：练习镜子 — 反映事实，不分配任务。

---

## 二、为什么这样选

### 2.1 为什么吉他优先

| 维度 | 吉他 | 钢琴 |
|------|------|------|
| 用户画像匹配 | 自学比例极高（>70%），最需要 AI 陪练 | 跟师比例高，自学少 |
| 收音可行性 | 吉他 + 耳机 + 手机麦 = 天然隔离 | 钢琴收音干扰大 |
| AB 循环价值 | riff/solo 片段循环是高频需求 | 更多整曲练习 |
| 和弦指法图价值 | 吉他和弦图是刚需 | 钢琴和弦相对标准 |
| AI Cover 适配 | 吉他翻唱是主流场景，音质自然 | 钢琴 AI Cover 质量差 |
| 社区氛围 | 云音乐吉他创作者活跃 | 钢琴内容偏古典/考级 |

### 2.2 为什么 Web 先行

| 维度 | Web | Native App |
|------|-----|------------|
| 开发效率 | 1 套代码全端覆盖 | iOS + Android 两套 |
| 发布迭代 | 即时更新无审核 | Store 审核周期 |
| 用户获取 | 链接分享即达，零安装 | 需下载安装 |
| 音频能力 | Web Audio API 完整支持变速变调、波形渲染 | 原生更强但 MVP 够用 |
| 收音能力 | MediaRecorder API 可用 | 原生更好 |
| 离线能力 | PWA + Service Worker | 天然支持 |

### 2.3 为什么用 AI Cover 做预置曲目

**核心问题**：原版音频版权谈判周期长、成本高，阻塞 MVP。

**AI Cover 方案**：
- 用 AI 翻唱工具生成目标歌曲翻唱版本
- 保留原曲和弦走向、曲式结构、BPM，音色/演唱是 AI 生成
- 练习场景用户关心「和弦走向 + 节奏 + 结构」，不是音色还原度
- MVP 内测阶段用于个人练习，风险可控；正式上线前需谈词曲授权

### 3.2 选曲标准

- 吉他初学热门曲 + 云音乐播放量高
- 10 首覆盖 Am-F-C-G / C-G-Am-F / G-D-Em-C / Em-C-G-D 等主流和弦走向
- 难度阶梯：★ × 4 首 + ★★ × 4 首 + ★★★ × 2 首
- 覆盖分解和弦 + 扫弦两种弹奏方式

### 3.3 音频处理管线

```
选定曲目
  ↓
AI Cover 生成（Suno / Udio / 自建）→ 完整翻唱音频
  ↓
人工质检（和弦准确性、节奏稳定性、音质）
  ↓
服务端离线处理：
  ├── Demucs 分轨 → 伴奏轨 + 人声轨
  ├── Essentia 和弦识别 → 时间轴和弦标注 JSON
  ├── BPM / 调性检测 → 元数据
  └── 波形峰值预计算 → 峰值数据 JSON
  ↓
资源上传 CDN/OSS
  ↓
前端按需加载
```

### 3.4 扩曲计划

验证跑通后按月新增 10-20 首，由用户投票决定曲目优先级。

### 5.2 波形可视化

**技术方案**：WaveSurfer.js

| 需求项 | 规格 | 优先级 |
|--------|------|--------|
| 波形渲染 | 完整歌曲波形，播放位置实时光标追踪 | P0 |
| AB 区间可视化 | AB 区间在波形上高亮，支持拖动锚点 | P0 |
| 和弦叠加 | 波形上方按时间轴叠加和弦名称标签 | P0 |
| 段落标记 | 前奏/主歌/副歌/桥段分割线 + 标签 | P1 |
| 缩放 | 双指缩放（移动端）/ 滚轮（桌面端） | P1 |
| 波形颜色 | 已播放着色 / 未播放浅色 / AB 区间品牌色高亮 | P0 |
| 点击定位 | 点击波形任意位置跳转播放 | P0 |

**与 AB 循环整合**：
- 波形上直接拖选设置 AB 区间（辅助方式）
- A/B 按钮实时打标仍保留（主方式）
- 波形 AB 区间与控制区双向同步
- 循环播放时光标在 AB 区间内往复

**性能优化**：
- 波形峰值数据服务端预计算，前端仅渲染
- 移动端默认简化波形，桌面端完整波形
- 懒加载可视区域

### 5.3 AB 循环（Web 端适配）

沿用 PRD v1.1 完整规格，以下为 Web 适配要点：

| 维度 | 规格 |
|------|------|
| 打标方式 | ① 播放中按 A/B 按钮实时打标<br>② 波形上拖选区间 |
| 键盘快捷键 | `A`=设A点, `B`=设B点, `Esc`=重置, `Space`=播放暂停 |
| 触屏适配 | 长按波形设 A 点，再次长按设 B 点；触摸拖动锚点 |
| 循环缓冲 | 默认 0.5s 静音；可选无缝循环 |
| 最小区间 | 2 秒，不足自动调整 + Toast |
| 持久化 | IndexedDB（按 song_id 绑定），登录用户同步云端 |
| 循环计数 | 波形下方显示 "Loop #N" |

### 5.4 变速 + 变调

**技术实现**：SoundTouch.js（WASM），变速不变调。

| 需求项 | 规格 |
|--------|------|
| 变速范围 | 25%-150%，步进 5%，默认 100% |
| 变调范围 | ±6 半音，步进 1 半音，默认原调 |
| 预设按钮 | 50% / 60% / 70% / 80% / 100% |
| 实时生效 | 调速当前循环内立即生效，不中断播放 |
| 变调折叠 | 默认折叠，点击展开 |
| 键盘快捷键 | `[`=减速5%, `]`=加速5%, `-`=降半音, `=`=升半音 |
| 与和弦联动 | 变调后和弦名称自动转调更新 |

### 5.5 和弦显示 + 吉他指法图

**和弦显示**：
- 波形上方时间轴标签 + 当前和弦大字高亮
- 数据来源：服务端预计算 JSON
- 变调后自动转调

**吉他指法图**：
- 点击和弦名称弹出标准六线和弦图
- 标注手指位置、根音、品位
- 预置 20+ 常用和弦（C/Am/G/Em/F/Dm/D/A/E/Bm 等）
- MVP 只做吉他指法图

### 5.6 分轨音量控制

| 需求项 | 规格 |
|--------|------|
| 轨道数 | 2 轨：人声 + 伴奏 |
| 控制方式 | 每轨独立音量滑块（0-100%） |
| 默认状态 | 全部 100%，用户主动调节 |
| 技术实现 | Web Audio API GainNode |
| 加载策略 | 默认加载混合音频；首次调节时按需加载分轨文件 |

### 5.7 AI 陪练 — 练习镜子（MVP 范围）

MVP 做镜子层前两个模块，基于规则引擎，不依赖开麦。

**模块 1：练习轨迹记录**

记录所有练习行为：进入/退出、AB 循环段落和次数、变速轨迹、暂停位置和时长。数据存 IndexedDB + 上报后端。

**模块 2：跨 Session 记忆 + 循环检测介入**

| 功能 | 触发条件 | 反馈内容 |
|------|---------|---------|
| 续练提示 | 再次打开同一首歌 | 「上次练到副歌 78% 速度，继续？」 |
| 循环卡壳检测 | 同段循环 >8 次 + 速度无变化 | 「这里练了 8 遍了，试试放慢到一半？」 |
| 稳定提速建议 | 连续 5 遍速度稳定 | 「稳了 5 遍，可以试快一点了。」 |
| 暂停关怀 | 暂停 >2 分钟 | 「继续？还是换一首？」 |
| 疲劳提醒 | 连续 >25 分钟未换段落 | 「练了挺久了，完整弹一遍感受下？」 |
| Session 总结 | 退出练习模式 | 「今天练了 22 分钟，副歌速度 70%→82%」 |

**UI 设计**：
- 轻量提示条（位于波形下方），非弹窗
- 可点击展开详情或忽略
- 干预强度分级：视觉提示 → 一句话 → 给选项 → 建议换法
- 克制使用高级干预，不打断心流

## 六、视觉设计 — Neon 方案

### 6.1 设计哲学

- **深空霓虹**：深黑底色 + 霓虹三色发光，营造沉浸式练习氛围
- **毛玻璃层叠**：半透明卡片 + backdrop-filter blur，信息层次清晰
- **克制发光**：glow 只用于交互焦点和关键数据，不滥用
- **暗色优先**：适合长时间练习场景，护眼且专注

### 6.2 色彩系统

### 6.6 发光效果体系

**静态 Glow**：
- 按钮 hover：`box-shadow: var(--glow-cyan)`
- 播放按钮常驻：`box-shadow: 0 0 24px rgba(0,229,255,0.25), 0 0 24px rgba(255,45,149,0.2)`
- 和弦指位圆点：SVG `feGaussianBlur` stdDeviation=3

**动画 Glow**：
- 播放指针脉冲：`cursorPulse` — opacity 1↔0.55，1.2s 循环
- AI 镜子呼吸：`glowBreath` — box-shadow 透明↔cyan 微光，3s 循环

**渐变文字**（歌名、大字和弦、速度值）：
```css
background: var(--grad);
```

## 七、埋点系统设计

### 7.1 架构：本地 + 上报

```
用户操作 → 事件生成
               ↓
         IndexedDB 本地存储（实时写入）
               ↓
         批量上报策略：
           ├── 每 30 秒批量上报一次（在线时）
           ├── Session 结束时全量上报
           ├── 网络恢复时补传离线事件
           └── 本地保留最近 30 天数据（超期自动清理）
               ↓
         后端事件管道（API 接收 → 消息队列 → 数据库）
```

**为什么本地 + 上报**：
- 本地存储确保离线场景不丢数据
- 批量上报减少网络请求频率
- 本地数据支撑 AI 陪练实时分析（镜子层不依赖网络）
- 上报数据用于全局产品分析和迭代

### 7.2 核心事件定义

**Session 级事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `practice_session_start` | 进入练习模式 | song_id, user_id, timestamp, device_type, browser |
| `practice_session_end` | 退出练习模式 | song_id, total_duration_ms, loop_count, max_speed, min_speed |

**AB 循环事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `ab_loop_set` | 完成 AB 打标 | song_id, start_sec, end_sec, method(button/wave_drag) |
| `ab_loop_cycle` | 完成一次循环 | song_id, ab_start, ab_end, duration_ms, loop_index |
| `ab_loop_adjust` | 调整锚点 | song_id, old_start, new_start, old_end, new_end |
| `ab_loop_reset` | 重置 AB | song_id, session_id, loop_count_before_reset |

**变速/变调事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `speed_changed` | 变速操作 | song_id, from_speed, to_speed, method(preset/step/input/keyboard) |
| `pitch_changed` | 变调操作 | song_id, from_pitch, to_pitch |

**波形交互事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `waveform_seek` | 点击波形跳转 | song_id, from_sec, to_sec |
| `waveform_zoom` | 缩放波形 | song_id, zoom_level, method(pinch/scroll) |

**分轨事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `track_volume_changed` | 调节分轨音量 | song_id, track(vocal/instrumental), from_volume, to_volume |

**和弦/指法事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `chord_diagram_opened` | 点击查看指法图 | song_id, chord_name, position_sec |

**AI 陪练事件**：

| 事件 | 触发 | 关键字段 |
|------|------|---------|
| `mirror_prompt_shown` | 镜子提示展示 | song_id, prompt_type, prompt_content |
| `mirror_prompt_action` | 用户响应提示 | song_id, prompt_type, action(accept/dismiss/ignore) |

## 九、AI 陪练详细设计

### 9.1 规则引擎（MVP）

所有镜子功能基于客户端规则引擎，不需要服务端 AI 模型。

```javascript
// 规则示例
const rules = [
  {
    name: "loop_stuck_detection",
    condition: (state) =>
      state.currentLoopCount > 8 &&
      state.speedChangeInLastNLoops(8) === 0,
    action: {
      type: "suggest_slow_down",
      message: "这里练了 {count} 遍了，速度没在提升。试试放慢到 {halfSpeed}%？",
      level: "suggestion"  // nudge → reflection → suggestion → active
    }
  },
  {
    name: "stable_speed_detection",
    condition: (state) =>
      state.consecutiveCleanLoops >= 5,
    action: {
      type: "suggest_speed_up",
      message: "稳了 5 遍，可以试快一点了。",
      level: "reflection"
    }
  },
  {
    name: "long_pause_detection",
    condition: (state) =>
      state.pauseDurationMs > 120000,
    action: {
      type: "check_in",
      message: "继续？还是换一首？",
      level: "nudge"
    }
  },
  {
    name: "fatigue_detection",
    condition: (state) =>
      state.sameSegmentDurationMs > 25 * 60 * 1000,
    action: {
      type: "suggest_full_play",
      message: "练了挺久了，要不要完整弹一遍感受下？",
      level: "suggestion"
    }
  }
];
```
