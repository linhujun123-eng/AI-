## 一、技术选型（最终确认）

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| **框架** | React 19 + TypeScript | 19.x | 生态成熟，TS 保证大型项目可维护 |
| **构建** | Vite 6 | 6.x | 极快 HMR，原生 ESM，PWA 插件成熟 |
| **路由** | React Router 7 | 7.x | 轻量，4 个页面足够 |
| **状态管理** | Zustand | 5.x | 极简，适合中小项目，TS 友好 |
| **样式** | CSS Modules + CSS Variables | — | Neon 主题 token 用 Variables，组件隔离用 Modules |
| **音频引擎** | Web Audio API (原生) | — | 变速变调、GainNode 分轨、实时控制 |
| **波形可视化** | WaveSurfer.js | 7.x | 开源，支持 region/marker/zoom，社区活跃 |
| **变速不变调** | SoundTouch.js (WASM) | — | 客户端实时变速，零服务端成本 |
| **本地存储** | IndexedDB (via idb) | — | 埋点、AB 记忆、练习记录本地持久化 |
| **PWA** | vite-plugin-pwa | — | Service Worker + Manifest 一键配置 |
| **代码质量** | ESLint + Prettier | — | 统一风格 |
| **测试** | Vitest + Testing Library | — | 与 Vite 同生态，快 |

### 为什么不用 Tailwind？

Neon 方案依赖 CSS Variables + 自定义 glow/blur/渐变效果，这些在 CSS Modules 中直接写更自然。Tailwind 对 `backdrop-filter`、复杂 `box-shadow`、SVG filter 等支持需要大量 `@apply` 和 `extend`，反而增加配置复杂度。CSS Modules + Variables 对 Neon 这套视觉语言是更直接的选择。

## 三、MVP 最小可用版本 — 功能范围

### 3.1 MVP 目标

> **一个用户可以打开网页，选一首歌，看着波形用 AB 循环反复练习某个片段，并能调节速度。**

这是产品价值验证的最小闭环。

### 3.2 MVP 包含的功能（第一个可用版本）

| # | 功能 | 说明 | 涉及组件 |
|---|------|------|---------|
| 1 | **曲目列表页** | 展示 10 首预置歌曲卡片，点击进入练习 | HomePage, SongCard |
| 2 | **音频播放** | 加载并播放预置曲目，播放/暂停/进度控制 | useAudioEngine, audio-engine.ts |
| 3 | **波形可视化** | 完整歌曲波形渲染，实时播放指针，点击定位 | WaveformPanel, useWaveform |
| 4 | **AB 循环** | A/B 按钮打标，波形区间高亮，循环播放，Loop 计数 | useABLoop, ab-loop-store |
| 5 | **变速（保持音调）** | 预设按钮 + ±5% 步进，实时生效 | SpeedControl, SoundTouch.js |
| 6 | **和弦显示** | 波形上方和弦标签 + 当前和弦大字高亮 | ChordLabels, ChordDisplay |
| 7 | **Neon 视觉** | 深空黑 + 三色霓虹 + 毛玻璃卡片 + 发光效果 | neon.css, GlassPanel, 各组件样式 |
| 8 | **键盘快捷键** | Space/A/B/Esc/[/]/←/→ | useKeyboardShortcuts |
| 9 | **响应式布局** | 桌面+移动端适配 | 各组件 CSS Modules |

### 3.3 MVP 明确不做的

| 功能 | 推迟原因 |
|------|---------|
| 变调 | 变速已验证核心价值，变调可后加 |
| 分轨音量 | 需加载额外音频文件，增加首屏复杂度 |
| AI 镜子提示 | 需积累练习数据才有意义 |
| 吉他指法图 | 和弦名称显示已够用，指法图可迭代加 |
| 练习记忆（跨 Session） | 需 IndexedDB 完整方案，MVP 先验证播放+循环 |
| 埋点系统 | MVP 先跑通功能，埋点第二轮加 |
| PWA 离线 | 先确保在线体验流畅 |
| 练习记录页 | 无数据可展示 |
| 设置页 | 无需配置项 |
| 波形拖选 AB | A/B 按钮打标已满足，拖选是增强 |
| 段落标签 | 需额外标注数据，后续加 |

### 3.4 MVP 数据依赖

MVP 只需 **1 首歌** 的完整数据即可开发验证（以《晴天》为例）：

| 文件 | 说明 | 准备方式 |
|------|------|---------|
| `mix.mp3` | 混合音频（3-5 分钟） | AI Cover 生成 或 临时用无版权音频 |
| `peaks.json` | 波形峰值数据 | 可用 WaveSurfer.js 自动生成，或服务端预计算 |
| `chords.json` | 和弦时间轴 `[{time, chord}]` | 手动标注 或 Essentia 预计算 |

> 开发阶段可以用一首无版权吉他曲替代，不阻塞开发。

### 3.5 MVP 技术架构（精简版）

```
┌───────────────────────────────────────────────┐
│              浏览器（React PWA）                │
│                                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│  │ 音频引擎  │  │ 波形渲染  │  │ AB 循环控制   │ │
│  │ WebAudio │  │ WaveSurfer│  │ Zustand     │ │
│  │ +SoundT. │  │ .js      │  │ Store       │ │
│  └──────────┘  └──────────┘  └──────────────┘ │
│                                                │
│  静态资源：public/audio/（mp3 + peaks + chords） │
└────────────────────────────────────────────────┘

后端：无（纯前端，Vercel 静态部署）
```

**MVP 不需要后端**。所有数据都是预置静态文件。

## 五、MVP 开发步骤（建议顺序）

```
Step 1: 项目脚手架
  └── Vite + React + TS 初始化
  └── Neon 全局样式（CSS Variables + reset + animations）
  └── GlassPanel / NeonButton / GradientText 基础组件
  └── React Router 路由骨架（/ + /practice/:songId）

Step 2: 音频播放
  └── Web Audio API 封装（加载、播放、暂停、seek）
  └── 播放状态 Store（Zustand）
  └── 播放/暂停按钮 + 进度时间显示

Step 3: 波形可视化
  └── WaveSurfer.js 集成
  └── Neon 风格波形配色（三色渐变 或 自定义渲染）
  └── 播放指针同步
  └── 点击定位

Step 4: AB 循环
  └── A/B 打标逻辑 + Store
  └── WaveSurfer Region 渲染（cyan 高亮）
  └── 循环播放实现（到 B 点跳回 A 点）
  └── Loop 计数显示

Step 5: 变速
  └── SoundTouch.js 集成
  └── 速度预设按钮 + 步进控制
  └── 实时变速不中断播放

Step 6: 和弦显示
  └── 加载 chords.json
  └── 波形上方和弦标签渲染
  └── 当前和弦实时高亮（大字）

Step 7: 整合 + 打磨
  └── 键盘快捷键
  └── 响应式适配（移动端）
  └── 首页曲目列表
  └── 整体交互打磨 + 动画细节
```

预估 MVP 总工时：**5-7 天**（一人全栈）

## 六、关键技术决策备忘

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|---------|------|
| 波形渲染 | WaveSurfer.js | 纯 Canvas 自绘 | WaveSurfer 自带 Region/Marker/Zoom，开发快；Neon 配色可通过配置项实现 |
| 变速引擎 | SoundTouch.js WASM | Tone.js | SoundTouch 专注变速变调，体积小；Tone.js 功能全但过重 |
| 状态管理 | Zustand | Redux / Jotai | 轻量、无 boilerplate、TS 类型安全 |
| 样式方案 | CSS Modules + Variables | Tailwind / styled-components | Neon 设计大量自定义 glow/blur，CSS Modules 最直接 |
| 本地存储 | idb (IndexedDB) | localStorage | 结构化数据、大容量、异步，适合埋点+练习记忆 |
| 音频格式 | MP3 | WAV / OGG | 兼容性最好，文件大小适中 |
| 部署 | Vercel | Netlify / CloudFlare Pages | 零配置、全球 CDN、免费额度足够 |
